# Opposite Call ── 一方その頃、どこかのだれか。

あなたと正反対の生活をする誰かと、**一度きり・77往復**だけ言葉を交わす通話の体験。
声は変換され、記録されず、同じ相手とは二度と繋がらない。
終わると、二人の声から**一輪の花**が咲く。

nanoca(株式会社ナノカ)の三部作の三作目です。

## この版について(たいせつなこと)

これは**体験プロトタイプ**です。相手は実在の人ではなく、あらかじめ書かれた物語(シナリオ)です。
実際に見知らぬ誰かと繋がるサービスは開発中で、その計画は別文書「AppStoreまでの地図」にあります。
このことは、アプリ内の「この体験と、データの扱いについて」にも明記しています。

## 動かし方

`index.html` ただ一つのファイルで完結しています。ビルドも、サーバーも、データベースも要りません。

- そのままブラウザで開けば動きます(iPhone は Safari 推奨・イヤホン推奨)
- GitHub Pages に置けば、URL を持つサービスになります(下記)

### GitHub Pages での公開手順(3歩)

1. GitHub でリポジトリを作る(Public)
2. この3ファイル(`index.html` / `README.md` / `LICENSE`)をアップロードする
3. リポジトリの **Settings → Pages → Branch: main → Save**。数分後、表示された URL が公開先です

## 安全とデータの設計(監査する方へ)

このプロトタイプは「記録しない」を機能ではなく**構造**として実装しています。

- **通話内容は録音・保存・送信されない。** 交わした言葉は画面を閉じると消える(サーバーが存在しない)
- **生の声は端末の外に出ない。** マイク入力は端末内で花の設計図の数値(エネルギー・ゆらぎ)に変換され、音声は破棄される
- **個人につながる情報の自動モザイク。** 電話番号・メール・SNS ID らしき文字列は表示前に伏せられる
- **端末の外に出るのは一つだけ。** 「設計図を保存」を選んだときのみ、設計図番号・花の系統・日付・プロフィール区分・拾った言葉の"数"が運営に届く。言葉の中身は届かない
- **非医療の明示と専門窓口への導線。** 全画面に「窓口」ボタン(いのちSOS ほか)を常設
- **アクセシビリティ。** 声が出ない人は「文字で話す」で通話でき、相手の言葉は常に字幕表示(耳が聞こえなくても使える)。声が無い場合の花は「言葉標本」(文の長さ・間・句読点)から科学的に生成される
- **決定論。** 花の生成に乱数(Math.random)を使わない。同じ声・同じ言葉からは同じ花が咲く
